Grep those Xargs with Sed

Based on the last Post about the Vim editor, I make reference of Macros. In that example, I talk about changing many files, by adding a HTML class to a specific HTML tag.

Using the q on Normal Mode, do the changes in the first file, that you wish to see in all other files that have the same HTML structure, and stop the Macro by clicking q again.

What Vim is doing in the background is in fact the same we can do with sed, xargs and grep commands, to make all changes in a recursive way and without opening all files using any text editor.

So we want to find the textbox class in the HTML files of our current directory. So I did this:
grep -rli 'class="textbox" * | xargs -i@ sed -i 's/class=textbox"/class="textbox black"/g' @

Within less than 3 seconds, I could change more than 300 HTML files, which is the number of posts that the main page have at 22/05/2026.

You can see, this box around the text area, which represent the limits of it. That is what the black HTML class does, in the CSS file.
