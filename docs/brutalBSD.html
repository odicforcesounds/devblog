Simplicity: "We gain Knowledge by reading small or big Books which sometimes make reference to other Books."

OpenBSD ask some key tips for new users, like me or like you. Always use the more option and append the number of the email, to read the emails you will get on OpenBSD system by default. Is a Golden tip. Else you will only be able to read the last lines of your emails. For the first Welcome Message email, you lose all the necessary details to start, while you have access to all necessary details to participate, by sending your hw.sensors values.

In case you use the correct option to read your emails, you figure out the OpenBSD creates a map of necessary steps and readings to start configuring the system, much better than any other BSD or Linux System.

Their Welcome Message email, start to explain the very basics of using the man pages in the default Kshell and ask the user to type man afterboot, to start reading the first book, of many other books that are in the same OpenLibrary, which is always Open, at any hour of any day and any night. Then it gives an examples on how to install new packages using their Binary package manager, some extra notes and we start reading book after book. Sometimes in a exaustive experience, since they always link the necessary Books that are related to the one we are reading. All makes sense and there is no Hidden or secret knowledge, like what happen with FreeBSD man pages, when I was stucked on creating a disk file to use the VMs. In the other hand, OpenBSD vm manual pages, have the option to create those disk-files and all steps that are needed are explained the best they can, since we have access to the /etc/examples/* example files that are not empty.

By reading the afterboot, you can read how everything is connected, how everything should work, what limitations exist, what security problems can happen, what can you do to prevent them, how to configure each service they have that are always Open, at any time of day or night.

To me as a old Linux user, I always dream about this kind of approach, where there is a guide to start, which is available offline, in the shell, trying to prevent reading external resources, that can be old and useless, with options that don't exist in the -current version, creating misconfigurations and other possible problems to those who just follow along the steps that some articles describe, while I cannot deny, many awesome articles on the web about OpenBSD that is very useful, since they connect many services running at the same time, giving us a better understanding of how many services can run at the same time in peace.

man security is just one of many books, that are refered in the Afterboot Book.

Not like any other BSD or Linux system, OpenBSD already try to secure your system, while you can hardener a bit more, even if you can update and patch your system when bugs or security issues are found and fixed with syspatch. If you think about Linux in general, You need to spend a lot of time to secure your system, while you need to understand how to configure a secure system without creating new problems on running some service, like what can happen with SELinux by being unable to login as root when testing and learning about that tecnology that looks simple but can be damn complex too. You are using a Firewall by default. You can create all kind of network.interfaces too. All tools that I knew from Linux, I can see a simple version on OpenBSD while being more rebust.
