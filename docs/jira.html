Describing the necessary documents

When building a project, organizing information is one of the most important keys to have a Enjoyable Coding Flow experience. On jira, you can organize your documents that describe the application you wish to build on details, while maybe your documents are only pointers that refer to some feature that you wish to use one your application, but it will change the behavior when you define the rules or objectives of application when used by users.

You can have a feature that add some image. But uploading Images can have several purposes. The feature is "being able to upload images" but the purpose can be, just to share, or to sell, or to rate, or all at the same time... and those are what makes "your" application different from many others that have the same feature. They are able to "upload images" but that application have their own purpose (just to save on the cloud), which will not be public and therefor there is no rate system, you cannot sell using that platform, etc..
