Learning to use Vim

The best option is always to install Vim, open and type :help. You will start with the basis, while you will see that documentation is big too. Using Vim can be speed up out work. For example, today I figure out that I wish to add a New HTML class into the HTML class="textbox". I just type vim * into the docs/ directory and all posts where there, in one single page. Then you ask: how can the editor open more than 300 files and we only see one? And the answer is buffers.

Some of the default values that most vi users like to setup on the .vimrc is the set number. To record a macro you need to press q in visual mode, create the sequence you need to do in many files and press q to stop recording. Use the . to repeat the last changes into all other files. Select buffers by using the command buffer number-of-buffer. For example :buffer 13 or the shortcut :b 13 will open the 13 file in buffer. Use :buffers to list them all.

Since in this real example, I only need to find the HTML class, I created a Macro to apply the same commands (select and add new content) within all files. In less than 1 minute, I already change everything.

Naturally I could do the same using awk and sed but that is more complex.
