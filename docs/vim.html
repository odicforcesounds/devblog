Learning to use Vim

The best option is always to install Vim, open and type :help. You will start with the basis, while you will see that documentation is big too. Using Vim can be speed up out work. For example, today I figure out that I wish to add a new HTML class into the HTML textbox class. I just type vim * into the docs/ directory and all posts where there, in one single page. Then you ask: how can the editor open more than 300 files and we only see one? And the answer is buffers.

Since in this real example, I only need to find the HTML class, I created a Macro to apply the same commands (select and add new content) within all files. In less than 1 minute, I already change everything.

Naturally I could do the same using awk and sed but that is more complex.
