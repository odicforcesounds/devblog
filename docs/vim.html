The best option is always to install Vim, open and type :help. You will start with the basis, while you will see that documentation is big too. Using Vim can be speed up our work. When opening a new file with Vim, type i to start writting and use ESC keyboard to go back to visual mode. When you type i you are using Vim in Insert Mode.

Type :w to save changes and :close to close a file or use :q to leave editor. You can use sequences of commands, like for example: :waq!. This command save by the option :w, :a means all files in session, and :q is to quit and leave editor. Sometimes we can open a Terminal or a file explorer inside Vim. By using :! we force leaving Vim editor.

Today I figure out that I wish to add a new HTML class into all posts of this blog that use the same CSS configuratios. I just type vim * into the docs/ directory and all posts where there, in one single page. Then you ask: how can the editor open more than 300 files and we only see one? And the answer is buffers.

The default values that most vi users like to setup on the configuration file located by default at ~/.vimrc is the set number option. To record a macro you need to press q in visual mode, to start and create the sequence you need to do in many files, and press q to stop recording. Use the . to repeat the last changes into all other files. Use the command :buffers by listing all files in buffers. For example :buffer 13 or the shortcut :b 13 will open the 13 file in buffer. You can use :bnext or bpreviews to jump between open files.

If you have a Screen that have big resolution, you can easly work in several files using the :vsplit option to open new files in a vertical direction and :split in a horizontal one. You can use tabs too. Use / in visual mode to find some text and use :%s/text-to-modify/modified-text/g to change all text that is needed to change.
