Some issue that was fixed

Years ago, I had a 32 bit box that I wish to give a life. So I add OpenBSD 32 bits. Yesterday I look to that machine again and see how old it was. OpenBSD 6.6. Few years without touching that machine. So I start the normal process of upgrading, which for OpenBSD is interesting to see.

sysupgrade

It will do everything it needs without asking you to guide the system to upgrade. I just fix the /etc/installurl that was pointing to some cloudflare mirror, without my upgraded versions. After reaching version 7.6 I got an error and I lost root access.

So I when into single mode, re-mount the partition to be able to write and add my user to the /etc/doas.conf file.

Reboot and Tarannnnnn.. I can execute root commands using doas. Problems solved. Just another upgrade to be updated.

In fact, having this issue looks really cool to me. I cannot access root, like no one else can. (secured) <3

I will pkg_add -u to be sure that all packages are correct, but I still have the problem blowing my mind. The pinsyscall mechanism is ruin my OpenBSD box, while is Okay. I can deal with a system without having direct root access. But the truth is that after updaing the packages I finnaly recover my access to the root user. Even if I could use doas su -l and access it again. Not everything is a problem and old machines are noisy but not a problem. Losing access to root is a problem, while is not. :)
