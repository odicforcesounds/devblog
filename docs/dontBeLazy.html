So Your final product can be about Cars and the server that will host your API code can have nothing there or some blank HTML page. Since you can use Linux you can setup the code VARIABLES for your tokens in Bash not in some .env file. You can setup a Reverse Proxy to create a Virtual Path for the Communication Node of Your API. Then, since you can be Free on Your creativity, you can setup for this example about cars, the Virtual Path in your Reverse Proxy settings as:

SuperCars.com/s/u/p/e/wheels/and/motors/api/

And your HTTP Application, can host nothing on that page.
Not even a index.html page. Nothing at all.
