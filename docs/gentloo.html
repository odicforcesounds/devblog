Because using Links to install Gentoo we cannot see Warnings or Alerts

Last Gentoo Linux Review I describe how I get confused when installing Gentoo from the Minimal ISO. Then I face some issues, which are having Systemd and OpenRC in the same page, while Warnings are not shown as warnings, and therefor some details are not clear enought for me.

Then, I start reading the Full Installation Alpha Version of their documentation and I realize that warnings were there, while the Systemd information too. So at this point, I don't wish to follow Gentoo Handbook, but build my own by taking notes of their Handbook notes. Is the difference between using Firefox or w3m/links/elinks/lynx Terminal Web-Browsers.

For example, I was complaining about Memory Usage at compilation Time, but they have a Warning saying that using MAKEOPTS="-j2" is the safest way to not use all memory that we have in our machine, which can run out of memory if we use a higher number of CPUs, while building something like Rust or GCC, our Machine will crash and forcing shutdown is necessary. "Is a Installation Skill that Kills Installation." :D

Another curiosity is the Sakaki (some Japanese user that share his installation process) dedicated to a possible exclusive Systemd Handbook with a lot of improvements, if we compare it to the Default and Official Gentoo Manual, while the first warning on the Top of the Page, tell us to Not Follow that Guide. Then I ask myself: why that Guide is still there anyway? I don't know, but if I assume that some details are "out of Date", or we update them or we remove the full out of date article to the "Archive" folder. I mean, I would hide it, to not confuse users that use "Links" to read the manual pages available on Gentoo websites.

Is just a noobie perspective, they do whatever they like to. But yeah, they could spend some minutes updating their Documentation or spend some seconds by moving that article to some "archive folder..." mv article archive/ ... Something that any Linux user can do in less than 1 single day.

Can you imagine how much time you will spend on compilations, just to have a "stable" version for a few days? I mean, in a older machine, using MAKEOPTS="-j3" at the end of updating Chromium a new version is already available, taking you another week to compile the new version and updating again after finishing the compilate of the last update. In that aspect is Insane (in the Brain).

So, in this Time consuming perpective, since we will compile our own personal Kernel with the modules and features that we wish to have, maybe a ACCEPT_KEYWORDS="amd64" stable releases is the best choice, since updates are a bit slower. Even then, I would update Gentoo only if is strictly necessary. Is like Slackware, updates are damn slow, can take a year or so, while they still using OpenRC only. So, never update Gentoo and naturally, never add too much keywords to the USE="X" on the /etc/portage/make.conf file. Keep it as clean as possible, since when updates are needed you will need to change some and at that point, you should only change them on /etc/portage/package.use/ folder.

Just remember to go for vacations for some days or weeks, while compilation process gets done, if we have some luck and something wrong happen, which is amazingly common on Gentoo after not updating for some Time.. after 1 month you will probably encounter some problem.. After one year.. :)
