Logic Name Implications: The Source is the Answer

Popular known as TUIs , [Terminal User Interface], applications can have several options in a menu style, which simplify what is complex.

I really like SUIs [Shell Users Interfaces]:
A Terminal is a Graphical interface to access the Shell. So, the name of TUIs is **Wrong**.

To install any Linux/BSD system, you always have some SUI [Shell User Interface] to guide you and simplify your work, asking the necessary options that you need to configure. A similar SUI is from make menuconfig when compiling the Linux Kernel.

Even if the above User Interfaces use a library called Dialog, SUIs are the best choice for minimal usage. Therefor, I had this idea where, Instead of using WUIs[Web User Interfaces] to manage Systems, Future Generations should use SUIs to configure the server in a fancy style.

I know nothing about coding, but in my ignorance I would say that developers, only need to add a graphical option to each option that any service running on that box, can have as an option. Even if you think that, with this approach users will not read the man page, in fact will simplify their work time, because they just need to select [x] the correct options for them, and select option [ar](Apply & Restart) Service.

With SUIs, connections are secured with SSH only.
