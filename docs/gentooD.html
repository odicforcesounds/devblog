Gentoo Handbook was really awesome few years ago, but these days is Horrible Reading and Discusting. I mean, they always favorite OpenRC systems while for years, the main Handbook was focused on OpenRC, today is more about SystemD than OpenRC. They could create a separated Handbook for each, but they mix all in one. In fact, it looks like they switch to SystemD and Ignore OpenRC. This is about my experience. Last days I was about to install Gentoo but I realize that the Handbook become so confusion that I could do better. Naturally I don't know much as they do, but yeah.. mixing both OpenRC and SystemD in one single Handbook is crazy, even more when we know that OpenRC didn't develop alternatives to SystemD and therefor even if we USE="-systemd" Flag, we will need to install some SystemD Libraries and software. So, at this point, Gentoo become Shit to me. Sadly, since in my opinion was for years, the best Distro that could manage much better than any other distro, Memory usage.

You know, if Gentoo cares more about organizing the documentation of it's own Handbook, it would prepare the documentation that would work, what ever the level of expertise the user is and then, add links to what is the alternative of the Main choice. So, they could create a Handbook for OpenRC and point the same steps for SystemD or to talk about which alternatives exist to install the Kernel, with the basic consideration (what ever is user choice, this method will work), but that is not so clear these days.
